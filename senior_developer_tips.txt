================================================================================
[WhereIsIt] 프로젝트 성공을 위한 시니어 개발자의 핵심 가이드 & 팁
================================================================================

당신의 10년 차 사수로서, 프로젝트 운영 중 꼭 기억해야 할 핵심 팁들을 정리해 드립니다.
이 내용을 잘 숙지하시면 안전하고 전문적인 개발자가 될 수 있습니다!

1. 보안 및 환경 설정 (Security & Config)
--------------------------------------------------------------------------------
□ .env 파일 관리 (비밀 금고)
   - API 키나 DB 비밀번호는 소스 코드에 직접 적지 말고 반드시 .env에 보관하세요.
   - GitHub 등에 올릴 때 이 파일은 공유되지 않도록 .gitignore에 등록되어 있어야 합니다. (현재 완료)

□ 브라우저 노출 주의 (프론트엔드 보안)
   - 현재 방식(VITE_ 사용)은 브라우저에서 키가 노출될 수 있는 구조입니다. 
   - 아주 중요한 대규모 프로젝트라면 중간에 Backend 서버를 두는 것이 정석입니다.

□ .gitignore의 중요성
   - node_modules, dist, .env 등은 절대로 소스 저장소(Git)에 올리지 않습니다.
   - 프로젝트가 무거워지거나 보안 데이터가 유출되는 것을 막아줍니다.

2. 데이터 관리 및 인프라 (Data & Infra)
--------------------------------------------------------------------------------
□ Supabase 연동 (클라우드 저장소)
   - 앱 새로고침 후에도 데이터가 남으려면 Supabase SQL을 반드시 실행해야 합니다.
   - RLS(Row Level Security) 정책을 통해 내 데이터가 안전한지 항상 체크하세요.
   - 공개 개발 단계가 끝나면 'Public Access' 정책을 'Authenticated'로 변경하는 것이 좋습니다.

□ 이미지 최적화 (모바일 환경 고려)
   - 모바일 기기는 메모리가 제한적이므로 고화질 원본 사진은 앱을 느리게 만듭니다.
   - 현재 우리 앱은 400px 수준으로 자동 압축하도록 되어 있어 매우 효율적입니다.

3. AI 연동 가이드 (AI Integration)
--------------------------------------------------------------------------------
□ Gemini API 활용
   - AI는 질문(Prompt)에 따라 결과가 크게 달라집니다.
   - 한국어 서비스라면 프롬프트도 한국어로 명확하게 작성하면 더 정확한 결과가 나옵니다.
   - (현재 프롬프트는 한국어 최적화 튜닝을 완료해 두었습니다.)

4. 문제 해결 및 디버깅 (Troubleshooting)
--------------------------------------------------------------------------------
□ ERR_CONNECTION_REFUSED 해결
   - 이 에러는 개발 서버(npm run dev)가 꺼져 있을 때 발생합니다. 당황하지 말고 터미널에서 서버를 다시 켜주세요.

□ F12 개발자 도구 활용
   - 네트워킹 오류가 나거나 AI 응답이 안 오면 브라우저의 F12 -> Network 탭을 보세요. 
   - 에러의 진짜 원인(403, 500 등)을 가장 빨리 찾을 수 있는 방법입니다.

5. 시니어의 마인드셋 (Developer Mindset)
--------------------------------------------------------------------------------
□ "동작만 하는 코드"보다 "안전한 코드"
   - 항상 방어적 코딩(Edge case 처리)을 하세요. 
   - 사용자가 이상한 값을 넣거나 인터넷이 끊겼을 때 앱이 뻗지 않게 처리하는 것이 실력입니다.

□ 공식 문서(Docs)와 친해지기
   - 누군가의 블로그 글보다 라이브러리/프레임워크의 공식 사이트를 먼저 믿으세요.
