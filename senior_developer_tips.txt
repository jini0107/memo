# 👨‍💻 시니어 개발자의 꿀팁 (Senior Developer Tips)

본 파일은 프로젝트 진행 중 발생하는 주요 노하우와 개발 팁을 기록하는 저장소입니다.

---

### [2026-02-25] 🛠️ 개발 환경 업데이트 및 모델 관리
1. **업데이트 전 체크리스트**: 개발 도구나 모델(Antigravity 등)을 업데이트할 때는 실행 중인 프로세스를 완전히 종료한 후 별도의 터미널에서 진행하는 것이 안전합니다.
2. **에러 메시지 기록**: 업데이트 중 에러가 발생하면 에러 로그를 캡처하거나 텍스트로 저장해두세요. 원인 파악에 결정적인 단서가 됩니다.
3. **환경 변수(.env) 백업**: 큰 규모의 업데이트나 Git 롤백을 수행하기 전에는 항상 `.env` 파일과 같은 로컬 설정 파일을 따로 복사해두는 습관을 들입시다.

### [2026-02-24] 📘 문서화와 개발 효율성
1. **초기 세팅 문서화의 중요성**: 프로젝트 초기 세팅(Node.js 버전, DB 스키마, 환경 변수 등)을 `README.md`나 `SETUP_GUIDE.md`에 명확히 기록하면 팀원의 온보딩 시간을 80% 이상 단축할 수 있습니다.
2. **효과적인 README 구성**: README는 프로젝트의 첫인상입니다. 설치 방법뿐만 아니라 '왜 이 프로젝트가 필요한지', '핵심 기능이 무엇인지'를 시각적으로(Badge, Screenshot 등) 보여주는 것이 좋습니다.
3. **효율적인 Git 사용법**: `Atomic Commit`(하나의 커밋엔 하나의 작업만)을 생활화하세요. 문제가 생겼을 때 특정 시점으로 쉽고 빠르게 되돌릴 수 있습니다.

### [2026-02-20] 🔐 보안 및 UX 디자인
1. **방어적 UI 설계**: 비밀번호나 PIN 입력 시 잘못된 접근에 대해 단순히 '실패'라고만 표시하기보다, 점진적인 잠금(Lockout) 기능을 통해 보안성과 사용자 경험을 동시에 챙길 수 있습니다.
2. **상태 관리의 정밀함**: React에서 모달 창 등을 구현할 때 `key` 속성을 활용하면 컴포넌트를 물리적으로 다시 그리게 되어(Unmount/Remount), 이전 상태가 남는 부작용을 원천 차단할 수 있습니다.
3. **시각적 피드백**: 보안이 중요한 데이터는 단순히 숨기는 것뿐만 아니라, 자물쇠 아이콘이나 블러(Blur) 처리를 통해 '현재 보호되고 있음'을 직관적으로 알려주는 것이 중요합니다.
