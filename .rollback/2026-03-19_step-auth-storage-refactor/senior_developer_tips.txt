# Senior Developer Tips

[2026-03-19] 데이터 모델(Item), DB schema, 백업 포맷은 항상 같은 계약을 유지해야 합니다. 한쪽에만 필드를 추가하면 동기화 순간에 데이터가 유실됩니다.
[2026-03-19] Base64 이미지를 state/localStorage/DB에 직접 저장하면 용량과 성능 문제가 빠르게 커집니다. 업로드 파일은 Storage에 두고 메타데이터만 DB에 저장하는 구조가 기본입니다.
[2026-03-19] 보안 기능(PIN, 잠금, 시크릿 아이템)은 UI보다 저장 전략이 더 중요합니다. 브라우저 localStorage 평문 저장은 데모 수준을 넘기기 어렵습니다.
[2026-03-19] 보안 정보를 브라우저에 저장해야 한다면 최소 기준은 평문 저장 금지입니다. 해시 저장은 완전한 보안이 아니라 노출 피해를 줄이는 1차 방어선으로 봐야 합니다.
[2026-03-19] 레거시 데이터를 새 포맷으로 바꿀 때는 한 번에 끊기보다, 검증 성공 시 자동 마이그레이션하는 방식이 실제 서비스에서 장애를 줄입니다.
[2026-03-19] App.tsx가 800라인을 넘기기 시작하면 화면 컴포넌트가 아니라 애플리케이션 조립 지점으로 역할을 줄여야 합니다. 상태 훅과 도메인 로직을 분리하면 테스트와 회귀 대응이 쉬워집니다.
[2026-02-25] README는 설치 방법만 적는 문서가 아니라 프로젝트의 목적, 핵심 기능, 실행 방법을 한 번에 이해시키는 진입점입니다.
[2026-02-20] 보안 UI는 실패 메시지뿐 아니라 잠금, 힌트, 시각적 마스킹까지 함께 설계해야 실제 사용자 경험과 방어력이 올라갑니다.
