
# [기획서] 개인 자산 위치 관리 솔루션 "WhereIsIt"

## 1. 4W 프레임워크 문제 정의
- **Who (누가):** 바쁜 직장인, 정리 정돈이 서툰 사람, 다량의 취미 용품 소지자, 중요한 서류를 가끔 사용하는 시니어 등.
- **What (무엇을):** 여권, 인감증명서, 계절 가전 부속품, 특정 클라우드 폴더 경로, 비정기적 사용 비상금 등.
- **When (언제):** 물건을 보관한 지 오래되어 기억이 가물가물할 때, 혹은 급하게 해당 물건을 찾아야 하는 응급 상황 시.
- **Where (어디에서):** 옷장 깊숙한 곳, 창고 박스 내부, 디지털 클라우드의 하위 폴더 등 일상적이지 않은 장소.

## 2. 핵심 문제 정의
> "중요한 물건/정보를 Routine(일상적 장소)이 아닌 Non-routine(특이 장소)에 보관할 때 발생하는 '기억의 공백'이 검색 비용(시간/스트레스)을 발생시킨다."

## 3. 앱의 목적
> "사용자의 공간 및 정보 위치를 디지털로 기록하여, 물리적 탐색 시간 없이 즉각적인 위치 확인을 지원하는 개인용 위치 어시스턴트."

## 4. 기능 정의 (MVP 기준)
### 포함 기능 (Included)
- **항목 기록:** 물건 이름, 카테고리, 태그 입력 및 사진 첨부(선택).
- **위치 관리:** 위치 계층 구조(방 > 가구 > 서랍) 설정 기능.
- **검색 시스템:** 키워드 기반의 빠른 검색 및 AI 기반 자연어 검색 지원.
- **카테고리 자동화:** AI를 이용한 물건 이름 기반 카테고리/태그 추천.

### 제외 기능 (Excluded - v2 예정)
- **AI 객체 인식:** 사진 촬영 시 자동 물건 이름 추출.
- **QR 코드 관리:** 박스/서랍별 QR 코드 출력 및 스캔 연동.
- **다중 사용자 공유:** 가족 구성원 간 위치 정보 실시간 공유.

## 5. UX 흐름 및 화면 구성
1. **메인 화면 (Dashboard):** 
   - 검색 바 (최상단)
   - 등록된 물건 목록 (최신순/카테고리순)
   - 빠른 등록 FAB (Floating Action Button)
2. **등록 화면 (Registration):**
   - 물건 기본 정보 입력
   - 위치 트리 선택 (Selector)
   - AI 추천 버튼 (자동 태깅)
3. **검색 결과 화면 (Search):**
   - 키워드 일치 항목 나열
   - AI 응답 섹션 (위치 경로 강조)

## 6. 데이터 구조 (Technical)
```typescript
interface Item {
  id: string;          // 고유 식별자
  name: string;        // 물건명 (Required)
  locationId: string;  // 위치 ID 연동
  category: string;    // 카테고리 (Enum)
  tags: string[];      // 검색용 태그 리스트
  imageUrl?: string;   // 시각적 확인용 (Base64 or URL)
  updatedAt: number;   // 마지막 기록 시점
}

interface Location {
  id: string;
  name: string;
  parentId?: string;   // 계층 구조용 부모 ID
}
```
